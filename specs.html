Homelab Architecture & Components

Core Tools

  • Splunk – SIEM: Collects and analyzes logs from all lab machines. I use it to detect suspicious activity, visualize security events, and simulate real SOC dashboards.
  • Wazuh – EDR: Monitors endpoints for malware, policy violations, and abnormal behavior. It complements Splunk by providing real-time alerts and threat intelligence.
  • pfSense – Firewall: Provides network segmentation, NAT, and packet filtering. I use it to create a realistic network environment with DMZs and restricted segments.
  • VMware Workstation: Hosts all virtual machines, allowing me to simulate attackers, victims, and monitoring servers in a controlled environment.

Virtual Machines

  • Kali Linux – Attacker: Used to perform penetration tests, simulate attacks like brute-force and phishing, and test detection rules in Splunk and Wazuh.
  • Ubuntu Server – Victim: Receives simulated attacks, generates logs, and acts as a target for monitoring, giving a realistic environment for log analysis.
  • Wazuh Server: Central endpoint monitoring server that collects alerts from VMs, performs correlation, and provides insight into potential threats.
  • Splunk Server: Central SIEM server that ingests logs from all VMs, visualizes them in dashboards, and allows me to create custom detection rules for attacks.

Goals & Workflow

This lab simulates a real SOC environment. My goal is to generate attacks, detect them using Splunk and Wazuh, analyze logs, and understand the entire attack-to-response lifecycle.

Each component works together: pfSense isolates networks, VMs simulate attacks and targets, Wazuh monitors endpoints, and Splunk visualizes events and alerts for actionable insights.

By iteratively updating the lab weekly, I can test new attack vectors, refine detection rules, and document progress, demonstrating hands-on expertise to recruiters.
