// network topology

Three isolated subnets, one firewall, full visibility.

Every machine sits behind pfSense on its own segment. Attacker traffic is quarantined to its own subnet, and all logs converge on Splunk in the SOC zone. Nothing touches the host network except pfSense’s NAT uplink.

LAN — corporate SOC — monitoring ATK — attacker pfSense / IDS → logs to SIEM ⇢ attack path

The five machines

Each VM has a fixed role and a fixed address. Status reflects the live build.

The log pipeline

Four sources feed the SIEM. Splunk normalizes them and runs detections on top.

Windows Security

Logon events, account changes, and Kerberos service tickets (4624/4625/4769) from DC01 and the endpoint.

Sysmon

Process creation, network connections, and image loads — the high-fidelity endpoint telemetry behind most detections.

pfSense firewall

Allowed and blocked flows across subnets, exported as syslog to spot scanning and lateral movement.

Suricata IDS

Signature alerts on the wire — port scans, exploit attempts, and known-bad traffic patterns.
