How the lab came together, phase by phase.

A running record of what I built, what broke, and how I fixed it. The messy parts — like migrating the whole lab between hypervisors mid-build — are the parts that taught me the most.

phase 01

Network segmentation

done

Stood up pfSense as the core of the lab and carved the network into three isolated subnets, each on its own interface:

  • LAN 10.0.1.0/24 — corporate segment for AD and endpoints
  • SOC 10.0.2.0/24 — monitoring segment for Splunk and Nessus
  • ATK 10.0.3.0/24 — quarantined attacker segment for Kali

DHCP disabled on the internal nets so addressing stays deterministic. The only path to the internet is pfSense’s NAT uplink — the attacker subnet can never reach the host network.

phase 02

Active Directory domain

done

Built DC01 on Windows Server 2022 and promoted it to a domain controller for shivenlab.local — running AD DS, DNS, and DHCP. Then populated it like a small org:

  • Organizational units for IT and Marketing
  • Domain users (jsmith, slee, itadmin) with itadmin in Domain Admins
  • A planted SPN on jsmith to act as Kerberoasting bait

This gives the attacks something realistic to chew on — real accounts, real group nesting, real misconfigurations to find.

phase 03

Hypervisor migration — VirtualBox → VMware

done

The host’s Hyper-V components conflicted with VirtualBox and blocked the Windows 11 endpoint from booting. Rather than fight it, I moved the entire lab to VMware Workstation — without rebuilding anything.

// the fix that worked

OVF export choked on a disk-capacity mismatch, and a direct attach tried to mount the disk as raw physical storage. The clean path was converting each disk explicitly: VBoxManage clonemedium for the DC and vmware-vdiskmanager -r for pfSense, then attaching the converted VMDKs to fresh VM shells.

Result: pfSense came back with all four interfaces intact, and DC01 booted with the full domain preserved — Get-ADDomain confirmed every user, OU, and role survived. Only fix needed was re-setting the static IP after the virtual NIC changed.

phase 04

Windows endpoint + telemetry

in dev

Building WKSTN01 (Windows 11), joining it to shivenlab.local, and instrumenting it for visibility:

  • Sysmon with a tuned config for process, network, and image-load events
  • Splunk Universal Forwarder shipping Security and Sysmon logs to the SIEM
  • Script-block logging enabled to capture PowerShell activity
phase 05

Splunk SIEM & detections

in dev

Standing up Splunk in the SOC subnet, onboarding all four log sources, and turning the planned detection set into live correlation searches with notable events and dashboards.

phase 06

Attack validation + Suricata

planned

Run each attack from Kali end-to-end, confirm the matching detection fires, and tune out false positives. Add the Suricata IDS feed from pfSense for network-layer coverage.

phase 07

Vulnerability management

planned

Run authenticated Nessus scans across the lab, then prioritize remediation by CVSS and map findings to NIST 800-53 controls — closing the loop from detection to hardening.
