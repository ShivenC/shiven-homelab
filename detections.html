// detection engineering

Each detection has an attack that proves it works.

Detection content is only real if you can trigger it on demand. For every technique below I pair the Splunk search that catches it with the exact Kali command that sets it off — the same attack-then-detect loop a SOC uses to validate coverage.

note: The searches and triggers below are the planned detection set. They’re written and version-controlled now; the “in dev” tag on each clears once it’s validated end-to-end against live telemetry in the running lab.

The detections

SPL on the right, the trigger on the left. Severity reflects impact in the lab’s context.

01

Password brute force against the domain

T1110 medium
trigger · kali
# spray a wordlist at SMB
crackmapexec smb 10.0.1.10 \
  -u jsmith \
  -p rockyou.txt
splunk search
index=wineventlog EventCode=4625
| bucket _time span=5m
| stats count by _time, Account_Name, src
| where count > 10
Counts failed logons (4625) per account in 5-minute windows. More than ten failures from one source is the classic spray signature — a real user fat-fingering a password doesn’t fail ten times in five minutes.
02

Kerberoasting — service ticket harvesting

T1558.003 high
trigger · kali
# request all SPN tickets
impacket-GetUserSPNs \
  shivenlab.local/jsmith \
  -dc-ip 10.0.1.10 -request
splunk search
index=wineventlog EventCode=4769
  Ticket_Encryption_Type=0x17
| stats count by Account_Name, Service_Name
| where count > 3
Service-ticket requests (4769) using weak RC4 encryption (0x17) are the tell. A planted SPN on jsmith acts as bait — one account suddenly pulling many service tickets is someone collecting hashes to crack offline.
03

LSASS memory access — credential dumping

T1003.001 high
trigger · endpoint
# dump lsass with mimikatz
mimikatz # sekurlsa::logonpasswords
splunk search
index=sysmon EventCode=10
  TargetImage="*lsass.exe"
  GrantedAccess=0x1010
| stats count by SourceImage, Computer
Sysmon Event 10 logs process access. Anything opening lsass.exe with 0x1010 rights is reading credential memory — legitimate software almost never does this, so it’s a high-signal alert.
04

Suspicious encoded PowerShell

T1059.001 medium
trigger · endpoint
# obfuscated download cradle
powershell -enc \
  SQBFAFgAKABJA...# base64
splunk search
index=sysmon EventCode=1
  Image="*powershell.exe"
| regex CommandLine="(?i)-enc|-nop|iex|downloadstring"
| table _time, Computer, CommandLine
Sysmon Event 1 captures process creation with full command lines. Encoded (-enc), no-profile (-nop), and in-memory execution (IEX) flags are hallmarks of download cradles and fileless payloads.
05

Lateral movement over admin shares

T1021.002 high
trigger · kali
# exec across hosts with creds
impacket-psexec \
  shivenlab.local/itadmin@10.0.1.20
splunk search
index=wineventlog EventCode=4624
  Logon_Type=3
| stats dc(Computer) as hosts by Account_Name
| where hosts > 2
Network logons (4624, type 3) fanning out from one account to several machines in a short window is the footprint of an attacker reusing stolen credentials to hop across the domain.
